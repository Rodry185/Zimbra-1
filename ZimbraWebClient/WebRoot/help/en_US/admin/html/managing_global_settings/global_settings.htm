Managing Global Settings

Global Settings define the default global values for servers, accounts, COS, and domains. These default values and parameters apply if the values and parameters have not been explicitly defined in settings configured elsewhere.

The defaults are configured during installation. You can change the settings at any time from Global Settings.  Global settings include tabs to configure the following:

  • General to set the global ceiling for the number of results from a GAL search, to define the default domain, and to configure the number of threads that can be used to get the content from remote data sources. See General Information for Global Settings.

  • Attachments to enable rules to reject messages that include attachments of a specific extension, to disable attachments from being read, and to convert attachments to HTML for viewing. See Blocking Attachments to Email.

  • MTA to enable authentication, set maximum message size, enable or disable protocol and DNS checks, and add X-Originating-IP to message headers. See Configuring the MTA Tab for Global Settings.

  • POP to enable POPS3 service. See Configuring POP.

  • IMAP to enable IMAP service. See Configuring IMAP.

  • AS/AV to set anti-spam and anti-virus rules. See Anti-Spam Protection and Anti-Virus Protection.

  • Free/Busy Interop to set up free/busy scheduling across a mix of Zimbra servers and third party email server. See Making Free/Busy View Available.

  • Themes  to customize the color scheme of existing themes and to add your logo to the themes. See How to customize themes and logos.

  • Briefcase. To configure the company name that is displayed in the prompt on the Authentication Required dialog used to log into Briefcase folders shared with external guests. The default is Zimbra.

  • S/MIME. (Secure/Multipurpose Internet Mail Extensions) To set up external LDAP server S/MIME settings.  Multiple servers can be set up.

  • License to update and install your Zimbra license and to see the current license information. See Updating Your Zimbra License.

  • HSM (hierarchical storage management), to establish how old a message should be before it is moved to a secondary volume. See Hierarchical Storage Management.

  • Backup/Restore. You can set the backup mode, where backup sessions should be saved and the email address that should be notified about the results of the backup. Note: Throttle automatic backups is used when migrating accounts to the ZCSwhen the Auto-Grouped mode is selected. See the Administrator's Guide, Backup chapter.

  • ACL. The ACL (Access Control List) tab is associated with delegated administration rights granted on the selected target. The ACL tab displays a list of Access Control Entries (ACE) granted on that current target. You can add, edit or delete an ACE from this tab.
