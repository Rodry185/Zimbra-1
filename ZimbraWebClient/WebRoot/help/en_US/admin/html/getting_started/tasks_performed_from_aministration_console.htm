Global Administrator's Tasks from Admin Console

The global administrator manages the complete Zimbra Collaboration Server email system. From the administration console you can:

  • Provision and manage end user accounts

  • Use the migration tool to move accounts to ZCS

  • Create and maintain distribution lists

  • Create delegated administrators

  • Manage domains and servers

  • Configure global settings for security, attachments, MTA, POP, IMAP

  • Monitor servers status and performance statistics

  • Create different Class of Service (COS) types that can be assigned to accounts to define enabled features and settings

  • Create password policies

  • Manage volumes for message store and index store and configure a secondary volume for HSM

  • Monitor cluster services

  • Manage, view and install Certificates

  • Moving a mailbox

  • Schedule backup sessions and restore from backups

Some management tasks are not available from the console. Refer to the Zimbra Administration Guide for details about all other administration tasks.
