Установка параметров для окна создания сообщения

На вкладке «Настройки>Создание сообщения» можно найти следующие варианты.

  • Написать письмо. Выберите текстовый редактор по умолчанию: для HTML или для обычного текста.

Установите параметры шрифта для использования по умолчанию при создании сообщений, документов и записей ежедневника в формате HTML.  Можно определить стиль, размер и цвет шрифта, используемые по умолчанию.

  • Ответ/пересылка в формате исходного сообщения. Даже если формат используемого по умолчанию текстового редактора отличается от формата полученного сообщения, установите этот флажок, чтобы ответное сообщение было в том же формате, что и полученное.  Это необходимо для того, чтобы получатель смог получить сообщение в наиболее пригодном для чтения формате.

  • При настройке Всегда создавать письмо в новом окне для создания сообщения открывается новое окно, отдельное отдельно от почтового ящика Zimbra. По умолчанию открывается вкладка создания письма на панели инструментов приложения. В обоих случаях при открытом окне создания сообщения можно просматривать почтовый ящик и перемещаться в нем.

  • Обязательно проверять правописание перед отправкой сообщения. Установите этот параметр для автоматической проверки правописания перед отправкой сообщения.  При обнаружении ошибок в диалоговом окне указывается количество ошибок и предлагается исправить их перед отправкой.  

  • Создаваемые сообщения автоматически сохраняются как черновики каждые пять минут. Если автоматически сохранять черновики не нужно, эту функцию можно выключить. Можно сохранить сообщение как черновик вручную.

  • Ответить/Ответить всем.  Укажите, нужно ли включать исходное сообщение в тело ответа. Можно включать или не включать текст исходного сообщения, включать его как вложение, включать в тело письма с выбранным префиксом, включать только самое последнее сообщение.  (Другими словами, в кавычки будет заключен только текст, написанный отправителем сообщения, но не предыдущий текст.)

  • Переслать. Укажите, нужно ли включать исходное сообщение в тело пересылаемого сообщения. Можно выбрать вариант «Включить исходное сообщение с заголовками и префиксом» или «Включить исходное сообщение как вложение».

  • Префикс. Укажите, нужно ли ставить префикс > или | перед каждой цитируемой строкой предыдущих сообщений, которые пересылаются или на которые дается ответ.

  • Отправленные сообщения. По умолчанию включен параметр Сохранить копию в папке «Отправленные». Копии отправленных сообщений сохраняются в папке «Отправленные». Если это не нужно, можно установить флажок Не сохранять отправленные сообщения.
